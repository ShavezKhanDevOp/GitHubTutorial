Git cheat sheet for all

Configer the git by using User name:

  • git config --global user.name `User Name`
  • git config --global user.email `User Email`

Check config is completed or not using:

  • git config --global user.name
  • git config --global user.email

Initialize the git:

  • git init

Check the status:

  • git status
  • git status -s (give short status)

Add the files in staging area:

  • git add `file-name` (for single file to add in stage)
  • git add -A (Add only newly added files to stage area)
  • git add . (for all file\'s to add in stage)

Commit the changes:

  • git commit
  • git commit -m "message"

Create the files:

  • touch `fileName with extensions`

Checkout the old commits:

  • git checkout `fileName` (recover the single file)
  • git checkout -f (recover the all the file)

Check the logs:

  • git log
  • git log -p -`number of changes` (i.e git log -p 1)

Compare the working tree:

  • git diff (working tree with staging area)
  • git diff --staged (staged tree with last commit)

Skip stage and add commit code directly :

  • git commit -a -m "Message"

Remove the files:

  • git rm `file_name` (remove the file from stage as well as directory)
  • git rm --cached `file_name` (remove the file from stage only)

Add git ignore file and add file/folder name which we have to ignore.

  • touch .gitignore

format to add file name in gitignore

  • myLogs.log --> this get ignore from entire project
  • /myLogs.log --> get ignored from directory where .gitignore file is present
  • *.log --> ignore all file has extension log
  • folderName/ --> ignore this folder.

Show branches:

  • git branch

Create new branch:

  • git branch `branch_name`

Point to new branch:

  • git checkout `branch_name`
  • git checkout -b `branch_name` (create and switch to new branch)

Merge the new branch in master:

  • git merge `branch_name`

Working with remote repositories:

Add remote rempository:

  • git remote add origin https://github.com/{UserName}/{RepositoryName}.git

Check the remote details:

  • git remote

Push the branch to remote branch:

  • git push origin {branchName}
  • git push -u origin {branchName} (for sure push)

Clone the repository:

  • git clone {repo Url}
